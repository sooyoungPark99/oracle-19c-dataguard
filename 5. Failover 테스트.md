# Failover 테스트

### 실습 전 주의사항

> Failover는 되돌리기 어려운 작업이므로 반드시 VM 스냅샷을 먼저 찍는다.
> 스냅샷 이름 예시: failover전, failover전2

| VM | 현재 역할 | DB 이름 |
|----|---------|---------|
| ora19c | Primary | PROD |
| sbdb | Standby (mount + 복구모드) | SBDB |

---

### 1. Standby DB 복구 모드 준비

```sql
shutdown immediate;
startup mount;

alter database recover managed standby database disconnect from session;

select process, status from v$managed_standby;
-- MRP0 WAIT_FOR_LOG 상태이면 정상이다.
```

---

### 2. 실습 1 - SQL 명령어로 수동 Failover

#### 1. Primary 장애 시뮬레이션 (ora19c VM)

```sql
connect / as sysdba
shutdown abort;
```

#### 2. Managed Recovery 취소 (sbdb VM)

```sql
alter database recover managed standby database cancel;
```

#### 3. 최종 Redo 적용

```sql
alter database recover managed standby database finish;
```

> 데이터 손실을 감수하고 강제 완료할 경우 finish force 옵션을 사용한다.

#### 4. Standby를 Primary로 활성화

```sql
alter database activate standby database;
alter database open;
```

#### 5. 역할 확인

```sql
select database_role, open_mode from v$database;
-- PRIMARY / READ WRITE
```

---

### 3. 실습 2 - DGMGRL 브로커로 Failover

스냅샷을 failover전2로 복원한 후 시작한다.

#### 1. listener.ora 정적 서비스 등록 (PROD/SBDB 양쪽)

PROD listener.ora:

```
SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = PROD)
      (ORACLE_HOME = /u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = PROD)
    )
    (SID_DESC =
      (GLOBAL_DBNAME = PROD_DGMGRL)
      (ORACLE_HOME = /u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = PROD)
    )
  )
```

SBDB listener.ora:

```
SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = SBDB)
      (ORACLE_HOME = /u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = SBDB)
    )
    (SID_DESC =
      (GLOBAL_DBNAME = SBDB_DGMGRL)
      (ORACLE_HOME = /u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = SBDB)
    )
  )
```

#### 2. Broker 파라미터 설정 (PROD/SBDB 양쪽)

```sql
alter system set log_archive_config='DG_CONFIG=(PROD,SBDB)' scope=both;

alter system set dg_broker_config_file1='/home/oracle/PROD/dg_broker1.dat' scope=both;
alter system set dg_broker_config_file2='/home/oracle/PROD/dg_broker2.dat' scope=both;

alter system set dg_broker_start=true scope=both;

show parameter dg_broker;
```

#### 3. DMON 프로세스 확인

```bash
ps -ef | grep dmon | grep -v grep
# ora_dmon_PROD, ora_dmon_SBDB 확인
```

#### 4. DGMGRL 접속 및 Configuration 생성 (PROD에서)

```bash
dgmgrl sys/oracle@PROD
```

```
create configuration 'DG_PROD' as
  primary database is 'PROD'
  connect identifier is PROD;

add database 'SBDB' as
  connect identifier is SBDB
  maintained as physical;

enable configuration;
```

#### 5. 구성 상태 확인

```
show configuration;
show database verbose 'SBDB';
```

> Configuration Status가 SUCCESS이고 Transport/Apply Lag가 0 seconds이면 정상이다.

#### 6. DGMGRL로 Failover 실행

Failover는 살아있는 쪽(Standby)에 접속해서 수행한다.

```bash
dgmgrl sys/oracle@SBDB
```

```
failover to 'SBDB';
show configuration;
```

> PROD는 disabled + ORA-16661 상태가 된다. 복귀하려면 Reinstate 작업이 필요하다.

#### 7. 새 Primary(SBDB) 역할 확인

```sql
select name, db_unique_name, database_role, open_mode
from v$database;
-- SBDB / PRIMARY / READ WRITE
```
