# DML 작업 동기화 테스트

### 환경 준비 (터미널 4개)

| 터미널 | 역할 |
|--------|------|
| 1 | Primary (PROD) - scott |
| 2 | Primary (PROD) - sysdba |
| 3 | Standby (SBDB) - sysdba |
| 4 | Standby (SBDB) - scott |

---

### 1. Primary에서 기본 데이터 준비

```sql
prod
sqlplus scott/tiger

select count(*) from emp;
select empno, ename, sal from emp where empno = 7788;
```

---

### 2. Primary에서 DML 작업 수행

```sql
-- INSERT
insert into emp (empno, ename, job, sal, deptno)
values (9001, 'TESTUSER1', 'CLERK', 1500, 10);

insert into emp (empno, ename, job, sal, deptno)
values (9002, 'TESTUSER2', 'ANALYST', 3500, 20);

-- UPDATE
update emp set sal = 3500 where empno = 7788;

-- DELETE
delete from emp where empno = 7369;

commit;

select count(*) from emp;
select empno, ename, sal from emp where empno in (7788, 9001, 9002);
```

---

### 3. 로그 스위치 강제 발생 (Primary)

```sql
connect / as sysdba

alter system switch logfile;
alter system archive log current;

select max(sequence#) from v$archived_log;
```

---

### 4. Standby MRP 상태 확인

```sql
sbdb
sqlplus / as sysdba

select process, status from v$managed_standby;
```

> MRP0가 APPLYING_LOG 또는 WAIT_FOR_LOG 상태이면 정상이다.

---

### 5. Standby를 Read Only로 열어 동기화 확인

```sql
alter database recover managed standby database cancel;
alter database open read only;

connect scott/tiger

select count(*) from emp;
select empno, ename, sal from emp where empno in (7788, 9001, 9002);
select empno, ename from emp where empno = 7369;
```

> Primary에서 DELETE한 7369번 사원이 Standby에서도 조회되지 않으면 동기화 성공이다.

---

### 6. Standby를 다시 복구 모드로 전환

```sql
connect / as sysdba

shutdown immediate;
startup mount;

recover managed standby database disconnect;
select process, status from v$managed_standby;
```
