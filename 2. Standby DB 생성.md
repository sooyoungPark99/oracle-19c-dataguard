# Standby DB 생성 (SBDB)

### 1. Standby 디렉터리 준비

Primary에서 복사해오는 파일을 받을 경로를 준비한다.

```bash
mkdir -p /home/oracle/SBDB
```

---

### 2. Primary 전달 대상 파일 경로 확인

어떤 경로를 복사할지 사전에 확정한다.

```sql
-- datafile
select file_name from dba_data_files;

-- tempfile
select file_name from dba_temp_files;

-- redo log file
select member from v$logfile;

-- controlfile
select name from v$controlfile;
```

> Primary에서 Standby로 전달하는 파일: datafile, tempfile, redo log file
> controlfile은 이후 단계에서 standby controlfile을 별도로 생성하여 교체한다.

---

### 3. Primary DB 정상 종료

파일 복사 중 변경을 막아 복사본 일관성을 확보한다.

```sql
shutdown immediate;
exit;
```


### 7. Standby Controlfile 생성

PROD를 mount로 올리고 standby controlfile을 생성한다.

```sql
startup mount;
alter database create standby controlfile as '/home/oracle/physical.ctl' reuse;
exit;
```

생성한 controlfile을 SBDB 경로에 복사한다.

```bash
cp /home/oracle/physical.ctl /home/oracle/SBDB/disk1/ctrl1.ctl
cp /home/oracle/physical.ctl /home/oracle/SBDB/disk2/ctrl2.ctl
cp /home/oracle/physical.ctl /home/oracle/SBDB/disk3/ctrl3.ctl
```

### 10. initSBDB.ora 생성

```bash
sbdb
cd $ORACLE_HOME/dbs
vi initSBDB.ora
```

```
compatible=19.3.0.0
control_files=(/home/oracle/SBDB/disk1/ctrl1.ctl,
               /home/oracle/SBDB/disk2/ctrl2.ctl,
               /home/oracle/SBDB/disk3/ctrl3.ctl)
db_block_size=8192
db_name=PROD
service_names=SBDB
global_names=true
job_queue_processes=10
open_cursors=500
processes=100
remote_login_passwordfile='EXCLUSIVE'
memory_target=1G
undo_management='AUTO'
undo_tablespace='UNDOTBS1'
db_recovery_file_dest_size=4G
db_recovery_file_dest=/home/oracle/SBDB/flash
db_unique_name=SBDB
standby_file_management=auto
db_file_name_convert='/u01/app/oracle/oradata/PROD','/home/oracle/SBDB'
log_file_name_convert='/u01/app/oracle/oradata/PROD','/home/oracle/SBDB'
log_archive_dest_1='location=/home/oracle/SBDB/arch valid_for=(all_logfiles,all_roles)'
log_archive_dest_2='service=PROD LGWR SYNC AFFIRM valid_for=(online_logfiles,primary_role)'
fal_server=PROD
fal_client=SBDB
```

| 파라미터 | 의미 |
|---------|------|
| db_name=PROD | Primary와 동일해야 한다 |
| db_unique_name=SBDB | Data Guard에서 고유 식별자로 사용된다 |
| db_file_name_convert | Primary 경로를 Standby 경로로 변환한다 |
| standby_file_management=auto | Primary 파일 추가 시 자동으로 반영된다 |
| fal_server/fal_client | 아카이브 누락 시 자동 수신에 사용된다 |

### 11. Standby Redo Log 생성 (SBDB)

```sql
startup pfile=$ORACLE_HOME/dbs/initSBDB.ora;

alter database add standby logfile
  group 10 ('/home/oracle/SBDB/disk1/stby_redo10.log') size 100m,
  group 11 ('/home/oracle/SBDB/disk1/stby_redo11.log') size 100m,
  group 12 ('/home/oracle/SBDB/disk1/stby_redo12.log') size 100m,
  group 13 ('/home/oracle/SBDB/disk1/stby_redo13.log') size 100m,
  group 14 ('/home/oracle/SBDB/disk1/stby_redo14.log') size 100m,
  group 15 ('/home/oracle/SBDB/disk1/stby_redo15.log') size 100m;

select group#, bytes/1024/1024 from v$standby_log;
```

### 14. MRP 기동 및 동기화 확인

Standby에서 MRP를 기동한다.

```sql
alter database recover managed standby database
  using current logfile disconnect from session;

select process, status, sequence#
from v$managed_standby;
```

> MRP0가 APPLYING_LOG 상태이면 정상이다.

Primary에서 변경을 발생시켜 Standby 반영을 확인한다.

```sql
-- PROD에서
create tablespace ts01
  datafile '/u01/app/oracle/oradata/PROD/disk4/ts01.dbf' size 10m;

alter system switch logfile;
alter system switch logfile;
alter system switch logfile;
```

```sql
-- SBDB에서
select name from v$datafile;
-- /home/oracle/SBDB/.../ts01.dbf 생성 여부를 확인한다.
```
