# Primary DB 생성 (PROD)

### 1. ORACLE_SID를 PROD로 설정

.bash_profile에서 SID를 PROD로 일괄 변경한다.

```bash
:%s/orcl/PROD/g
source .bash_profile
echo $ORACLE_SID
```

---

### 2. 데이터베이스 파일 디렉터리 생성

disk1~disk5 디렉터리를 만들어 data/control/redo를 분산 배치한다.

```bash
mkdir -p /u01/app/oracle/oradata/PROD
cd /u01/app/oracle/oradata/PROD
mkdir disk1 disk2 disk3 disk4 disk5
ls
```

> -p 옵션을 사용하면 하위 디렉터리까지 한 번에 생성된다.

---

### 3. PFILE(initPROD.ora) 작성 후 NOMOUNT 기동

$ORACLE_HOME/dbs에 initPROD.ora를 생성한다.
컨트롤파일 경로는 disk1~disk3으로 3중화한다.

```bash
cd $ORACLE_HOME/dbs
vi initPROD.ora
```

```
db_name=PROD
compatible=19.3.0
memory_target=1G
processes=300
undo_management=AUTO
undo_tablespace=UNDOTBS1
remote_login_passwordfile=EXCLUSIVE
control_files=('/u01/app/oracle/oradata/PROD/disk1/ctrl1.ctl',
               '/u01/app/oracle/oradata/PROD/disk2/ctrl2.ctl',
               '/u01/app/oracle/oradata/PROD/disk3/ctrl3.ctl')
db_unique_name=PROD
standby_file_management=auto
db_file_name_convert='/home/oracle/SBDB','/u01/app/oracle/oradata/PROD'
log_file_name_convert='/home/oracle/SBDB','/u01/app/oracle/oradata/PROD'
log_archive_dest_1='location=/home/oracle/PROD/arch valid_for=(all_logfiles,all_roles)'
log_archive_dest_2='service=SBDB LGWR SYNC AFFIRM valid_for=(online_logfiles,primary_role)'
fal_server=SBDB
fal_client=PROD
```

```bash
sqlplus / as sysdba
startup nomount pfile=$ORACLE_HOME/dbs/initPROD.ora
select instance_name from v$instance;
```

### 5. CREATE DATABASE 수행

수동 생성 시 핵심 파일인 datafile/controlfile/redo logfile이 생성된다.

```sql
create database PROD
user sys identified by oracle
user system identified by oracle
datafile '/u01/app/oracle/oradata/PROD/disk1/system01.dbf'
size 100M autoextend on maxsize unlimited extent management local
sysaux
    datafile '/u01/app/oracle/oradata/PROD/disk2/sysaux01.dbf'
    size 50M autoextend on maxsize unlimited
default temporary tablespace temp
    tempfile '/u01/app/oracle/oradata/PROD/disk3/temp01.dbf'
    size 50M autoextend on maxsize unlimited
undo tablespace undotbs1
    datafile '/u01/app/oracle/oradata/PROD/disk4/undotbs01.dbf'
    size 50M autoextend on maxsize unlimited
logfile
    group 1 ('/u01/app/oracle/oradata/PROD/disk4/redoG1M1.rdo',
             '/u01/app/oracle/oradata/PROD/disk5/redoG1M2.rdo') size 100M,
    group 2 ('/u01/app/oracle/oradata/PROD/disk4/redoG2M1.rdo',
             '/u01/app/oracle/oradata/PROD/disk5/redoG2M2.rdo') size 100M,
    group 3 ('/u01/app/oracle/oradata/PROD/disk4/redoG3M1.rdo',
             '/u01/app/oracle/oradata/PROD/disk5/redoG3M2.rdo') size 100M,
    group 4 ('/u01/app/oracle/oradata/PROD/disk4/redoG4M1.rdo',
             '/u01/app/oracle/oradata/PROD/disk5/redoG4M2.rdo') size 100M,
    group 5 ('/u01/app/oracle/oradata/PROD/disk4/redoG5M1.rdo',
             '/u01/app/oracle/oradata/PROD/disk5/redoG5M2.rdo') size 100M;

select status from v$instance;
```

### 8. listener.ora 정적 등록 및 tnsnames.ora 설정

기존 listener.ora를 백업하고 정적 서비스 등록으로 변경한다.

```bash
cd $ORACLE_HOME/network/admin
vi listener.ora
```

```
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.23.217)(PORT = 1521))
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (ORACLE_HOME = /u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = PROD)
    )
  )
```

```bash
vi tnsnames.ora
```

```
PROD =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.23.217)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = PROD)
    )
  )

SBDB =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.43.217)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = SBDB)
    )
  )
```

```bash

lsnrctl start
sqlplus scott/tiger@prod
```

### 9. 패스워드 파일 생성 및 Standby로 복사

Data Guard 구성에서 패스워드 파일이 반드시 필요하므로 별도로 생성하고 복사한다.

```bash
cd $ORACLE_HOME/dbs
orapwd file=orapwPROD password=oracle format=12 force=y
cp orapwPROD orapwSBDB
scp orapwSBDB oracle@192.168.43.217:$ORACLE_HOME/dbs/
```

### 10. Archivelog 모드 전환

Standby로 redo를 전달하려면 Primary가 아카이브 로그를 생성할 수 있어야 한다.

```sql
shutdown immediate;
startup mount;
alter database archivelog;
alter database open;
archive log list;
```

> alter database archivelog; 는 mount 상태에서만 실행 가능하다.

---

### 11. Force Logging 활성화

Primary에서 발생한 변경사항이 반드시 redo로 남아 Standby로 전달되게 한다.

```sql
alter database force logging;

alter database add supplemental log data
(primary key, unique index) columns;

alter system archive log current;
alter system switch logfile;

select force_logging from v$database;
```
