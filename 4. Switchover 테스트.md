# Switchover 테스트

### VM 기동 순서

> Primary가 완전히 OPEN된 후 Standby를 올려야 아카이브 갭 없이 동기화된다.

#### Primary (PROD) 기동

```sql
lsnrctl start

sqlplus / as sysdba
startup;

select instance_name, status from v$instance;

select dest_id, status, target
from v$archive_dest
where target = 'STANDBY';
```

#### Standby (SBDB) 기동

```sql
lsnrctl start

sqlplus / as sysdba
startup mount;

recover managed standby database disconnect;

select process, status from v$managed_standby;
select open_mode, database_role from v$database;
```

---

### 1. 현재 역할 확인

```sql
-- PROD
select open_mode, database_role from v$database;
select count(*) from scott.emp;

-- SBDB
select open_mode, database_role from v$database;
```

---

### 2. Phase 1 - PROD를 Standby로 전환

#### Switchover 가능 여부 확인

```sql
select switchover_status from v$database;
-- TO STANDBY: 가능
```

#### 최종 아카이브 로그 전송

```sql
alter system archive log current;
alter system switch logfile;
```

#### PROD를 Physical Standby로 전환

```sql
alter database commit to switchover to physical standby with session shutdown;
```

#### PROD를 MOUNT로 올려 역할 확인

```sql
startup mount;
select open_mode, database_role from v$database;
-- MOUNTED / PHYSICAL STANDBY
```

---

### 3. Phase 2 - SBDB를 Primary로 전환

#### Switchover 가능 여부 확인

```sql
select switchover_status from v$database;
-- TO PRIMARY
```

#### Managed Recovery 취소

```sql
alter database recover managed standby database cancel;
```

#### SBDB를 Primary로 전환

```sql
alter database commit to switchover to primary with session shutdown;
```

#### SBDB를 OPEN하여 역할 확인

```sql
shutdown immediate;
startup;
select open_mode, database_role from v$database;
-- READ WRITE / PRIMARY
```

---

### 4. Phase 3 - 새 Standby(PROD)에 MRP 재시작

```sql
recover managed standby database disconnect;
select process, status from v$managed_standby;
```

---

### 5. Phase 4 - 역할 전환 검증

#### 새 Primary(SBDB)에서 DML 수행

```sql
connect scott/tiger

insert into emp (empno, ename, job, sal, deptno)
values (9999, 'SWITCHTEST', 'MANAGER', 5000, 30);
commit;

select count(*) from emp;

connect / as sysdba
alter system switch logfile;
alter system archive log current;
```

#### 새 Standby(PROD)에서 동기화 확인

```sql
alter database recover managed standby database cancel;
alter database open read only;

connect scott/tiger
select count(*) from emp;
select empno, ename, job from emp where empno = 9999;
```

확인 후 복구 모드로 복귀한다.

```sql
connect / as sysdba
shutdown immediate;
startup mount;
recover managed standby database disconnect;
```

---

### 6. VM 종료 순서

#### Standby(PROD) 종료

```sql
sqlplus / as sysdba
alter database recover managed standby database cancel;
shutdown immediate;
```

```bash
lsnrctl stop
```

#### Primary(SBDB) 종료

```sql
sqlplus / as sysdba
shutdown immediate;
```

```bash
lsnrctl stop
sudo shutdown -h now
```
